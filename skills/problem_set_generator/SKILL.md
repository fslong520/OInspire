# 题单与训练计划生成器

## 何时使用

当学生提出以下需求时主动触发：
- "给我出一套练习题"
- "我想练练 XXX（知识点）"
- "接下来一周/一个月怎么安排训练"
- "我要参加 XX 比赛，需要准备"
- "我哪里薄弱，帮我补一补"

## 工作流程

### 0. 读取学生画像（如果存在）

**先读 `memory/student/` 下的文件，再生成题单：**

- `profile.md` — 了解学生画像、目标、薄弱点
- `knowledge_matrix.md` — 查看各知识点掌握状态
- `error_log.md` — 查看近期错题和高频错误模式
- `templates.md` — 查看模板熟练度，哪些需要巩固
- `contest_log.md` — 查看最近比赛表现和 upsolve 进度

**如果有历史数据**：基于画像直接生成个性化题单，不需要重新问基本信息。

**如果是第一次**：进入诊断阶段（步骤 1）。

### 1. 诊断阶段（关键）

**先了解学生情况，再生成题单：**

| 必问项 | 说明 |
|---------|------|
| 当前水平 | 入门 / 基础 / 提高 / 冲刺 |
| 编程语言 | C++ / Python / 其他 |
| 目标比赛 | NOIP普及组 / NOIP提高组 / CSP / 蓝桥杯 / 其他 |
| 可用时间 | 每天能投入多少小时 |
| 薄弱环节 | 哪些知识点不太熟 |
| 当前刷题量 | 已经刷过多少题 |

**话术示例：**
> "在出题单之前，先让我了解你——你现在学到哪个阶段了？每天能拿出多少时间？有没有特别想攻克的知识点？"

### 2. 生成题单

#### 题单格式

```markdown
## 📋 训练题单：{主题名称}

**目标：** {简述训练目的}
**难度梯度：** {入门→提高}
**预计用时：** {X 天 / X 题}

| # | 题目 | 难度 | 知识点 | 来源 | 建议用时 |
|---|------|------|--------|------|---------|
| 1 | [A+B Problem](链接) | ⭐ | 基础语法 | 洛谷 | 5min |
| 2 | ... | ... | ... | ... | ... |

### 训练建议
- 先易后难，不要跳过基础题
- 每道题独立思考至少 20 分钟再想看题解
- 写完后对比题解，学习更优雅的写法
- 遇到卡住的题，标记下来，隔天再做
```

#### 难度标注
- ⭐ 入门（语法基础，直接模拟）
- ⭐⭐ 基础（简单数据结构，基础算法）
- ⭐⭐⭐ 提高（经典算法，需要思维转换）
- ⭐⭐⭐⭐ 进阶（综合应用，多算法结合）
- ⭐⭐⭐⭐⭐ 冲刺（比赛真题级别）

#### 推荐题库来源
- **洛谷**（luogu.com.cn）：国内最全面的 OJ，适合各阶段
- **Codeforces**（codeforces.com）：全球最大竞技平台，适合提高
- **AtCoder**（atcoder.jp）：题目质量极高，适合思维训练
- **LeetCode**（leetcode.cn）：适合打基础，面试也常用
- **LibreOJ**（loj.ac）：适合深入学习经典算法

### 3. 生成训练计划

#### 训练计划格式

```markdown
## 📅 {周期}训练计划

**阶段目标：** {本阶段要达到的能力}
**时间跨度：** {X 周 / X 月}

### 第 1 周：{主题}
| 日期 | 内容 | 题单 | 目标 |
|------|------|------|------|
| Day 1 | 学习 {知识点} | 3-5 题 | 理解概念 |
| Day 2 | 练习 {知识点} | 5-8 题 | 熟练应用 |
| Day 3 | ... | ... | ... |
| Day 7 | 周总结 + 模拟赛 | 1 场 | 查漏补缺 |

### 周总结模板
- ✅ 掌握：{列出已掌握的}
- ⚠️ 薄弱：{列出仍需加强的}
- 🎯 下周重点：{调整方向}
```

#### 计划原则
- **循序渐进**：不要一上来就挑战难题
- **刻意练习**：每个知识点至少 10-15 题才能形成肌肉记忆
- **及时反馈**：每天记录做对率，每周复盘
- **劳逸结合**：每周至少一天休息或做趣味题

### 4. 持续跟踪

生成题单/计划后：
- 询问学生完成情况
- 根据反馈调整难度
- 记录薄弱环节，下次重点覆盖
- 适时给予鼓励和肯定

### 5. Upsolving 环节（赛后补题）

> **做过的题不复盘，等于白做。比赛完不补题，等于白比。**

每次模拟赛/正式赛后，生成 upsolving 计划：

```markdown
## 🔄 Upsolving 计划：{比赛名称}

**比赛日期：**
**成绩：** {得分}/{总题数}

### 补题清单
| 题号 | 题目 | 赛中状态 | 补题状态 | 完成日期 |
|------|------|---------|---------|---------|
| A | | ✅做出 | — | — |
| B | | ❌没做出来 | 🟡进行中 | |
| C | | ❌没做出来 | ⚪未开始 | |
| D | | ❌没看 | ⚪未开始 | |

### Upsolving 规则
- 没做出来的题，赛后要独立做出来
- 看题解后，必须自己重新写一遍（不是抄）
- 三天后重做，检验是否真正掌握
```

### 6. 模板训练计划

> **模板不是抄来的，是自己敲出来的。**

当学生需要巩固模板时：

```markdown
## 🔨 模板训练：{算法名称}

**目标：** 闭眼能写，肌肉记忆

### 训练步骤
1. 理解算法原理（不写代码，先讲清楚思路）
2. 照着模板敲一遍（理解每行代码的作用）
3. 空手写一遍（不看任何参考）
4. 限时写一遍（5 分钟内完成）
5. 变体写一遍（稍作修改的场景）

### 检验标准
- [ ] 能独立写出，无语法错误
- [ ] 5 分钟内完成
- [ ] 能解释每行代码的作用
- [ ] 能做简单变体
```

## 知识点地图（参考）

### 入门阶段
- 变量与数据类型
- 条件语句与循环
- 函数与递归基础
- 一维数组与字符串

### 基础阶段
- 二维数组与结构体
- 排序算法（冒泡、选择、插入、快速、归并）
- 基础数据结构（栈、队列、链表）
- 简单贪心与模拟
- 二分查找

### 提高阶段
- 动态规划入门（背包、区间、树形）
- 图论基础（BFS、DFS、最短路径）
- 树与二叉树
- 并查集
- 前缀和与差分

### 进阶阶段
- 线段树与树状数组
- 高级 DP（状态压缩、数位 DP）
- 图论进阶（拓扑排序、强连通分量、最小生成树）
- 字符串算法（KMP、Trie、AC 自动机）
- 数学（组合数学、数论基础）

### 冲刺阶段
- 网络流
- 字符串高级（后缀数组、后缀自动机）
- 平衡树
- 计算几何
- 博弈论

## 注意事项

- **不要给答案**：可以给题单和提示，但不直接给出题代码
- **因材施教**：根据学生水平调整难度，不要"一刀切"
- **留有余地**：题量不要排太满，留出消化时间
- **标注重点**：对特别有价值的题目，加粗或标注"必做"
- **推荐题解**：部分题目可以附上优质题解链接

## 记忆更新

生成题单/计划后，更新学生记忆：

1. **知识矩阵** — 将本次涉及的知识点状态更新
2. **错题本** — 学生后续完成的题目如有错误，记录到 `error_log.md`
3. **模板库** — 学生练习了新算法模板，更新 `templates.md` 熟练度
4. **比赛记录** — 学生参加比赛后，更新 `contest_log.md` 成绩和 upsolve 进度
5. **画像** — 如果对话中发现了学生新特点，更新 `profile.md`
6. **对话记录** — 简要记录本次生成的题单和学生反馈
