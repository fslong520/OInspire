# 学生记忆系统

> 用的越久，越懂这个孩子。

## 文件结构

```
memory/student/
├── profile.md           ← 学生画像（长期档案，持续更新）
├── knowledge_matrix.md  ← 知识掌握追踪矩阵（每个知识点的状态）
├── templates.md         ← 算法模板库（模板熟练度追踪）
├── error_log.md         ← 错题本（记录错误和原因）
├── contest_log.md       ← 比赛记录（成绩、upsolve、心态）
└── conversation_log.md  ← 对话记录模板（每次对话后填写）
```

## 更新规则

### 每次对话后必须做

1. **更新画像** (`profile.md`)：
   - 如果发现学生新的特点、优势、薄弱点 → 更新
   - 如果学生提到个人信息（年级、目标比赛等）→ 更新
   - 如果观察到能力变化 → 更新评估

2. **更新知识矩阵** (`knowledge_matrix.md`)：
   - 学生提到的知识点 → 检查状态是否需要更新
   - 学生完成的练习 → 更新对应知识点状态
   - 发现学生某知识点比预期好/差 → 调整

3. **记录错题** (`error_log.md`)：
   - 学生做错的题 → 记录题目、错误原因
   - 定期标记已掌握的错题 → 更新状态

4. **更新模板库** (`templates.md`)：
   - 学生练习了新算法模板 → 更新熟练度
   - 赛前检查 → 标记哪些模板还需要巩固

5. **记录比赛** (`contest_log.md`)：
   - 每场比赛后 → 记录成绩、心态、upsolve 进度
   - 赛后复盘 → 分析失误原因，制定改进计划

6. **填写对话记录** (`conversation_log.md`)：
   - 记录讨论的知识点、学生的困惑
   - 写下观察：理解速度、情绪状态、亮点
   - 规划下一步

### 长期累积效应

- **第 1 次对话**：知道学生名字和目标
- **第 5 次对话**：知道学生薄弱点和优势
- **第 20 次对话**：能预测学生在哪些题上会卡住
- **第 50 次对话**：比学生自己更了解他们的学习模式
- **第 100 次对话**：一个眼神就知道该给什么题

## 诊断逻辑

根据积累的数据，自动生成诊断：

```
if 同一知识点反复出错 → "这块需要重新学，不是练的问题"
if 正确率持续上升 → "你在进步，继续保持"
if 突然大量出错 → "是不是状态不好？休息一下"
if 长时间没提到某知识点 → "我们该复习一下之前学的了"
```

## 画像深度

随着对话增多，画像应该从"基本信息"进化到：

**基础层**（前几次对话就能知道）
- 姓名、年级、目标
- 当前水平

**特征层**（5-10 次对话后）
- 学习风格（视觉型？听觉型？）
- 性格特点（急躁？细心？好胜？）
- 优势薄弱分布

**模式层**（20+ 次对话后）
- 什么时候容易犯粗心的错误
- 什么类型的题目最能激发兴趣
- 什么样的解释方式最有效
- 遇到挫折时的反应模式

**预测层**（50+ 次对话后）
- 预判哪类题会卡住
- 预判比赛时的心态变化
- 预判学习曲线上的瓶颈期
